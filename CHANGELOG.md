# Changelog

All notable changes to this project are documented in this file.

The format is loosely based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [0.6.0] - 2026-05-06

### Added
- `--max-wait <duration>` flag — give up waiting after the deadline (e.g. `30s`, `5m`).
  Default `0` keeps the previous unlimited-wait behavior. On timeout the dependent
  command is **not** executed; exit code `124` (matching `timeout(1)`).
- `exitCodeTimeout` / `exitCodeInterrupted` now distinguish deadline expiration
  from signal cancellation via `errors.Is(ctx.Err(), context.DeadlineExceeded)`.

## [0.5.0] - 2026-05-06

### Added
- `cmd/root_test.go` — coverage for `newLogger` (verbose, default, JSON-to-file,
  un-openable file path). Coverage 78.8% → 93.8%.
- `.github/dependabot.yml` — weekly grouped updates for Go modules and GitHub
  Actions, prefixed with `chore(deps)` / `chore(ci)`.
- Shell completions (bash/zsh/fish) generated by Cobra and shipped both inside
  release archives and via Homebrew formula — `brew install` users get tab
  completion automatically.

### Notes
- The goreleaser `brews → homebrew_casks` deprecation warning is **not**
  addressed yet: switching changes the install command for end users
  (`brew install` → `brew install --cask`) and moves the formula from
  `Formula/` to `Casks/`. Postponed until a v1.0.0 release with a proper
  `tap_migrations.json` migration.

## [0.4.0] - 2026-05-06

### Added
- Unit tests for `cmd/watch`, `pkg/executor`, `pkg/waiter`. Coverage at 78.8%.
- `google/shlex` for `--command` parsing — single/double quotes, escapes and
  unbalanced-quote errors are now handled correctly. `sh -c 'exit 42'` finally
  works.
- Graceful shutdown via `signal.NotifyContext(SIGINT, SIGTERM)` plumbed through
  `Waiter.Wait` as `context.Context`. On interruption the dependent command
  is skipped; exit code `130` (`128 + SIGINT`).
- `Taskfile.yml` replacing the old `Makefile` (declarative tasks: `install-deps`,
  `fmt`, `vet`, `lint`, `test`, `build`, `run`, `release:snapshot`, `ci`).

### Changed
- `.golangci.yml` migrated to v2 schema; CI bumped to `golangci-lint v2.1.6` via
  `golangci-lint-action@v7`.

### Removed
- `Makefile` — replaced by `Taskfile.yml`.

## [0.3.0] - 2026-05-06

### Added
- `--verbose` / `-v` flag now logs every poll cycle, watch duration, and the
  dependent command's exit code (closes #8).
- `--logfile <path>` — JSON-formatted append-only file logging in parallel with
  the human-readable stderr stream (closes #9).
- `goreleaser` configuration with darwin/linux × amd64/arm64 matrix, GPG-signed
  `checksums.txt`, and automatic Homebrew formula publishing to
  `jtprogru/homebrew-tap`.
- `.github/workflows/release.yml` — triggered on tag push (`v*`), uses
  `GPG_PRIVATE_KEY` / `PASSPHRASE` / `HOMEBREW_TAP_GITHUB_TOKEN` secrets.
- `.github/workflows/build.yml` — replaced by goreleaser snapshot for PR
  validation.

### Fixed
- `cmd.Execute()` was running Cobra inside a goroutine (`go cobra.CheckErr(...)`),
  causing `main` to return before any command actually executed.
- `Version` is now a `var` (was `const`), so goreleaser's `-ldflags`
  `-X cmd.Version=...` actually overrides it. Added `Commit` and `BuildDate`.

## [0.2.1] - 2022-08-15

Initial release on the new release infrastructure (handcrafted GitHub Actions
workflow, `softprops/action-gh-release`).
