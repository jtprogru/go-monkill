## [1.0.1] - 2026-05-06

### Changed
- Build constraint `//go:build linux || darwin` added to `main.go`,
  formalising the supported-platform set already enforced by the CI and
  release matrix.
- README documents the supported-platform policy in the lead.

Documentation/build-policy change only; no functional changes to the CLI
surface or behaviour.

## [1.0.0] - 2026-05-06

First stable release. Marks the API and the CLI surface as stable; future
breaking changes will bump the major version.

### Changed
- **Homebrew distribution**: switched from formula to cask via goreleaser's
  `homebrew_casks` schema (the legacy `brews` is being phased out upstream).
  The cask installs the same prebuilt binary, removes the macOS quarantine
  xattr in a postflight hook so Gatekeeper doesn't refuse to launch the
  unsigned binary, and ships shell completions.

### Migration

If you installed via `brew install jtprogru/tap/go-monkill` (formula),
run once:

```shell
brew uninstall go-monkill
brew install --cask jtprogru/tap/go-monkill
```

The formula `Formula/go-monkill.rb` will be removed from the tap shortly
after v1.0.0 ships.

## [0.8.0] - 2026-05-06

### Added
- `run` subcommand: spawn a child process, capture its exit code, and
  optionally dispatch hooks via `--on-success`, `--on-failure`, `--on-exit`.
  The utility exits with the child's exit code; hook failures are logged
  but do not override it.
- Coverage 94.6% â†’ 95.6%.

### Notes
- This unblocks the `--on-success` / `--on-failure` semantics that
  `watch` cannot offer (kernel does not expose exit codes for non-child
  PIDs). `watch` for monitoring existing PIDs, `run` for orchestrating
  new ones.

## [0.7.0] - 2026-05-06

### Added
- Multi-PID monitoring: `--pid` accepts a comma-separated list or can be
  repeated. Example: `--pid 1234,5678` or `--pid 1234 --pid 5678`.
- `--wait-for all|any` flag to control aggregation: run the command once
  every PID has exited (`all`, default â€” backward-compatible for
  single-PID usage) or as soon as the first PID exits (`any`). When `any`
  fires, the remaining waits are cancelled via context.

### Changed
- `watcher` signature now accepts `[]int` for PIDs and a `waitFor` mode
  string. The interface for `Waiter.Wait` is unchanged.

## [0.6.0] - 2026-05-06

### Added
- `--max-wait <duration>` flag â€” give up waiting after the deadline (e.g. `30s`, `5m`).
  Default `0` keeps the previous unlimited-wait behavior. On timeout the dependent
  command is **not** executed; exit code `124` (matching `timeout(1)`).
- `exitCodeTimeout` / `exitCodeInterrupted` now distinguish deadline expiration
  from signal cancellation via `errors.Is(ctx.Err(), context.DeadlineExceeded)`.

## [0.4.0] - 2026-05-06

### Added
- Unit tests for `cmd/watch`, `pkg/executor`, `pkg/waiter`. Coverage at 78.8%.
- `google/shlex` for `--command` parsing â€” single/double quotes, escapes and
  unbalanced-quote errors are now handled correctly. `sh -c 'exit 42'` finally
  works.
- Graceful shutdown via `signal.NotifyContext(SIGINT, SIGTERM)` plumbed through
  `Waiter.Wait` as `context.Context`. On interruption the dependent command
  is skipped; exit code `130` (`128 + SIGINT`).
- `Taskfile.yml` replacing the old `Makefile` (declarative tasks: `install-deps`,
  `fmt`, `vet`, `lint`, `test`, `build`, `run`, `release:snapshot`, `ci`).

### Changed
- `.golangci.yml` migrated to v2 schema; CI bumped to `golangci-lint v2.1.6` via
  `golangci-lint-action@v7`.

### Removed
- `Makefile` â€” replaced by `Taskfile.yml`.

## [0.3.0] - 2026-05-06

### Added
- `--verbose` / `-v` flag now logs every poll cycle, watch duration, and the
  dependent command's exit code (closes #8).
- `--logfile <path>` â€” JSON-formatted append-only file logging in parallel with
  the human-readable stderr stream (closes #9).
- `goreleaser` configuration with darwin/linux Ă— amd64/arm64 matrix, GPG-signed
  `checksums.txt`, and automatic Homebrew formula publishing to
  `jtprogru/homebrew-tap`.
- `.github/workflows/release.yml` â€” triggered on tag push (`v*`), uses
  `GPG_PRIVATE_KEY` / `PASSPHRASE` / `HOMEBREW_TAP_GITHUB_TOKEN` secrets.
- `.github/workflows/build.yml` â€” replaced by goreleaser snapshot for PR
  validation.

### Fixed
- `cmd.Execute()` was running Cobra inside a goroutine (`go cobra.CheckErr(...)`),
  causing `main` to return before any command actually executed.
- `Version` is now a `var` (was `const`), so goreleaser's `-ldflags`
  `-X cmd.Version=...` actually overrides it. Added `Commit` and `BuildDate`.
